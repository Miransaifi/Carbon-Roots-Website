# Carbon Roots Website Builder

Build and maintain a deployable Carbon Roots website that is commercially useful, visually disciplined, and credible.

## Follow these principles

- Keep messaging specific, concise, and trustworthy.
- Prefer clarity over hype.
- Avoid fake testimonials, inflated claims, and guaranteed outcomes.
- Keep design premium but restrained (dark hero, light body sections, sage/gold accents).
- Keep implementation simple and maintainable.

## Work sequence

1. Review current files in `/website`.
2. Confirm or refine section structure for a one-page site.
3. Write or improve copy for business clarity and investment-readiness positioning.
4. Implement clean static HTML/CSS/JS unless a framework is explicitly needed.
5. Check responsiveness, hierarchy, readability, and CTA clarity.
6. Leave output deployable with organized files and no placeholders.

## Copy standards

- Write for non-technical decision-makers.
- Use careful carbon-market language.
- Emphasize ARR pre-feasibility, indicative sequestration modelling, financial viability screening, and risk/data-gap identification.
- Position Carbon Roots as expert-led, faster and more affordable than large consultancies, without overclaiming.

## Design standards

- Use strong spacing and clear visual hierarchy.
- Keep typography modern and legible.
- Use restrained accents only where they improve hierarchy.
- Keep interactions minimal and polished.

## Default file targets

- `/website/index.html`
- `/website/styles.css`
- `/website/script.js`
- `/website/assets/`

## Done criteria

- Navigation works.
- Site is responsive.
- Buttons and form elements are visually consistent.
- Copy is reviewed and commercially coherent.
- No lorem ipsum or fake social proof.
- Code is clean and deployment-ready.
